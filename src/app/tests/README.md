tests/

Breve descrição:

Local para testes unitários e de integração.

O que colocar aqui:
- Testes para `services/`, `repositories/` e rotas HTTP.
- Fixtures para banco de teste e mocks para integrações externas.

Boas práticas:
- Manter testes rápidos e independentes.
- Usar `pytest` e separar unitários de integracao.
